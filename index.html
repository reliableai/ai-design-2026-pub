Designing Large Scale AI Systems — CS Master Course

Spring 2026 · reliableai.github.io · GitHub repo

Course Description

This course provides a comprehensive introduction to designing, building, and evaluating large-scale AI systems. Students will gain hands-on experience building AI-powered applications and learn rigorous evaluation methodologies essential for deploying AI systems in production.

Learning Objectives

By the end of this course, you will:

  1. Build effective, reliable, maintainable AI systems at scale. Solve complex problems by decomposing them in a principled way. Ship a reliable AI feature into an existing product.
  2. Use AI to build AI. Leverage AI for development, evaluation, and system management. Understand what this unlocks and where the limits are.
  3. Iterate towards success — rather than optimizing in the dark. Master the science of monitoring, analysis, and estimation. Stand up an evaluation harness you can run in CI.
  4. Master the key conceptual abstractions. Build observability, cost controls, and incident-ready guardrails. Take lessons from software engineering past, but do so critically.
  5. Separate signal from hype. Critically review emerging practices and proposed standards, learning what to use and where.
  6. Think in systems. Understand which engineering principles carry forward and which new abstractions are essential. Know when not to use agents (and what to do instead).

Course Structure

The course spans 12 weeks with two 90-minute lessons per week (24 lessons total), organized into four phases:

Phase 1

Building AI Agents

Lessons 1–3

LLM API calls, mastering individual calls, review cycles, state, and judges.

Phase 2

Iterating Towards Success

Lessons 4–6

Monitoring, quality estimation, uncertainty quantification, systematic iteration.

Phase 3

Building Effective, Manageable Complex Systems. At Scale.

Lessons 7–11

Councils of agents, system design, key abstractions, cost controls, guardrails, frameworks.

Phase 4

Capstone Project

Lessons 12–24

Apply everything: specification, midterm, reviews, working sessions, presentations.

Course Policies

Attendance

Attendance is expected for all lessons. Phase 3 sessions are particularly important as they include project reviews, working sessions, and presentations.

Academic Integrity

All work submitted must be your own (or your team's for the project). You must use AI tools to assist with coding and learning, but you must understand and be able to explain all concepts in your submissions and demonstrate ability to manage your codebase, with AI (or without, if you prefer, but that's going to be hard).

Collaboration

The midterm exam is strictly individual. The course project may be part of a larger group effort — different people can build different pieces of the same system — but each student has specific individual responsibilities and scope, and is graded individually on their own contribution.

Office Hours

Office hours are held right after each class. Additional appointments can be scheduled via email.
